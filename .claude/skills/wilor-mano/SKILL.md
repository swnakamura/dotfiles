# WiLoR / HaMeR / MANO 取り扱い知見（プロジェクト横断）

手 MANO 推定（WiLoR・HaMeR 系）と MANO モデルを扱うときの再利用知見。特に**左手**で踏みやすい罠。
具体的な実装・検証は hoi プロジェクト（`preprocess/hand_recon/PLAN.md`、メモリ `wilor-lefthand-multiview`）。

## ★可視化の色規約（一般常識・必ず守る）
**左手 = 赤 / 右手 = 青**。手の可視化における一般的な慣習。新しい手可視化を書くときは必ずこの色で。
（OpenCV は BGR なので 左=`(70,70,235)` / 右=`(235,150,60)` 程度。RGB なら左≈`(235,70,70)`/右≈`(60,150,235)`。）

## ★WiLoR 前処理の必須2点（推定品質を直接左右する）
WiLoR/HaMeR は **歪み無しのピンホール画像で手が大きく写っている**ことを仮定する。生フレームを直接入れない。
1. **整流（必須）**: 魚眼KB/Brown 歪みを必ず除去してから入力する。各仮想ピンホール画素を実カメラモデル
   （kb=`cv2.fisheye.projectPoints` / brown=`cv2.projectPoints`+全dist）でソース画素へ逆写像→`cv2.remap`。
   仮想ピンホールは手方向 d を光軸に向ける（R_cv）。
2. **手がクロップの一定割合を占めるようにする**: f_v は「**整流後の透視像で手 bbox が ~1/margin を占める**」よう
   直接決める＝`f_v=(S/2)/(margin·tan(ang))`（ang=手の角半径）。`tan(ang·margin)` で角度を膨らませる素朴版は
   透視 tan 伸長を無視し、特に魚眼端で手が極小になる。**魚眼端対策**: bbox 角を画像内にクリップしてから
   unproject（画像外の角は ray が発散）＋ ang を上限キャップ（手は ~38° 以上に広がらない）。

## MANO 基礎
- 右手モデル（`MANO_RIGHT.pkl`）。**単位はメートル**（手の広がり ~0.17m）。
- 関節 21 = MANO 16 関節（`J_regressor @ verts`）+ 5 指先頂点（smplx `vertex_ids['mano']`、順 thumb,
  index, middle, ring, pinky → idx 16–20）。指先頂点 ID = `[744, 320, 443, 554, 671]`。
- `hand_pose` は 15 関節の**局所回転**（rotmat or axis-angle）、`global_orient` は root（手首）回転。
  `betas`(10) は形状。WiLoR の MANO は `mano(global_orient, hand_pose, betas, pose2rot=False)`（rotmat 入力）。
- pkl は **chumpy 依存**で素の python では読めない。chumpy のある env（WiLoR/hort の .venv）で
  `v_template/shapedirs/J_regressor` を npz に一度抽出して chumpy 非依存化すると楽。
- **ボーン構造を仮定しない**: 関節順序を当てるより、MANO 静止関節の **MST**（最小全域木）で
  ボーン(隣接)を自動決定すると安全。

## ★左手の扱い（HaMeR / WiLoR 共通）= 最大の罠
ネットワークは**右手モデルしか持たない**。左手は **画像を水平反転して右手推定器を回す**方式。
- 入力: クロップ画像を左右反転（`do_flip`、`datasets/utils.py`）。**パラメータは変換しない**。
- 推論出力: `R`(global_orient) / `theta`(hand_pose) は **raw（右手系・未変換）**。
  保存 verts/joints だけ x 反転（`v[:,0]*=-1`）、`pred_cam` の tx も反転。
- レンダ時は x 反転で面の法線が裏返るので `faces_left`（面の頂点順逆転）を使う。
- **「真の左手 MANO パラメータ」が欲しい場合**は、axis-angle の **y,z 符号反転**を global_orient と
  hand_pose の両方に掛ける（`[1::3]*=-1; [2::3]*=-1`）。これは rotmat では **`M R M`（M=diag(-1,1,1)）共役**に相当。
  beta は左右対称で無変換。MANO_LEFT を別途持つ実装（deephand `mano_hamer.py`）もある。

## ★多視点で左手を world に揃えるときの罠（hoi/test_capture_0605 で確定）
1. **`M θ M` で hand_pose をミラーすると解剖が壊れる**。キネマティック姿勢のミラーは各局所回転を
   独立に M 共役するのとは違う。**θ は raw のまま使い、レンダ時に canonical verts を x ミラー**するのが正解。
   左右差はレンダだけで吸収: `V_world = R_global · [左なら x ミラー] MANO_RIGHT(I, θ, β) + t`。
2. **クロップ整流回転 R_cv の入れ忘れ**。WiLoR の global_orient は（整流クロップに通したなら）整流クロップ系。
   world 整合に R_cv を掛け忘れると右手でも 25°、掛けると 9° に揃う。
   - 右: `go_world = R_W_cam · R_cv · Rc`
   - 左: `go_world = R_W_cam · R_cv · M · Rc · M`（M は **crop 系の x 反転**）
3. **chirality 汚染**: 3D 位置で左右トラックを作ると、WiLoR が**誤ラベルした検出（is_right が逆=未ミラー）**
   が混入し、回転平均を破壊（左 59°、フィルタ後 12°）。**`is_right==side` でフィルタ必須**。
   - 一発診断: 各カメラの world 関節の **chirality 符号（例: (j1-j0)×(j5-j0)·(j9-j0) の符号）が揃うか**。
     揃わない（+/− 混在）= 誤ラベル混入。
4. **単眼 global_orient は奥行き向きが視点間で不一致**（per-view 2D は合うのに 3D 向きが揃わない）。
   上記 1–3 を正せば視点平均で揃う（11–12°）。それでも不足なら多視点 2D 再投影 bundle。

## ★β の絶対スケール（手の大きさ）は単眼平均では決まらない（hoi/0605 で確定）
- **単眼 β は絶対スケールが原理的に曖昧**で MANO 平均（≈大きめ成人手, 手首→中指先 178mm）へ回帰する。
  per-view β を視点平均しても**全視点が同じバイアスを共有**するので **~11% 大きいまま**残る
  （2D再投影 MANO/実手 = median 1.106、全カメラ一様で出る。三角測量関節はノイズで大きさ判定に使えない＝
  Procrustes スケールもノイズで下振れするので**2D再投影比で測る**のが確実）。
- **直し方 = 多視点 β バンドル**: 各視点の 2D（整流クロップのピンホール＝歪み無し・微分可能）に
  world MANO 関節を最小二乗で当て、**大域 β（L/R 共有, MANO は鏡像対称）**を再同定。**単眼で不確かな
  深度は捨て 2D 方向のみ**使う。残差は **手元 mm（px/f_v・カメラ距離）** に正規化してから扱い（クロップ毎の
  ズーム/距離の偏りを除去）、**下位50%トリミング（least-trimmed-squares）**で遮蔽・誤ラベル関節を除外。
- **スケールは「足場」s を s0(≈0.91)→1 へアニール**しながら縮みを β に移譲し、最終 s=1＝純 MANO(β)。
  MANO の β は**等方スケールを表せない**が、小さい手は学習分布内の**形状空間の点**として表現できる
  （s は局所解回避の continuation スキャフォールドとしてのみ使い、最後は外す）。
- 同時に **per-frame global_orient（手首回転のみ・位置はピン留め）も最適化**すると融合向き残差も改善。
- **β は全フレーム一括フィットが安定**（トリミングが悪観測を除くので汚れず、高次 β モードまで拘束される。
  確信フレームのサブセットに絞ると高次モードが過適合気味に暴れる）。

## 効かなかった/やめた方針（再挑戦しない）
- **per-keypoint 三角測量 → ボーン長で β 推定**: 視点間の単眼形状不一致で各関節が ~15mm ジッタし、
  総ボーン長が 1.6 倍に膨張（手の最大広がり自体は正常＝スケール問題ではない）。三角測量は**位置(wrist)
  のみ**信頼し、β は上記**多視点バンドル**で決める（per-view 平均だけでは ~11% 過大）。
- **三角測量3D関節への Kabsch で global_orient**: 関節が不正確なので不適（「不正確と結論した物に合わせる」
  矛盾）。向きは param 融合＋多視点 2D バンドルで。

## 遮蔽（自己遮蔽）可視度の軽量算出
メッシュをラスタライズせず**点群深度テスト**で十分高速（~0.8ms/検出）: 各関節の 2D 近傍に投影される
頂点の最小 z と関節 z を比較。閾値は手スケール相対（単位非依存）。手首等の内部関節を遮蔽と誤判定しない
ため `behind_frac≈0.15`（手の厚み相当）。
